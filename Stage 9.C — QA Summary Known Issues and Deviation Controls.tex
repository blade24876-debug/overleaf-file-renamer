\item \textbf{ISS-08-002} — Underwater governance availability risk: lifeguard/spotter/pool constraints can make underwater standards unschedulable — Must Fix Pre-Deploy
\item \textbf{ISS-08-003} — Underwater admissibility misclassification risk: \textbf{ST-UW-ADMISS} can be treated as assumed “Yes” without documented prerequisites — Must Fix Pre-Deploy
\item \textbf{ISS-08-004} — Long ruck gate compounded dependency stack: multiple categories must be ready; missing any forces reslot and can block late-phase gates — Must Fix Pre-Deploy
\item \textbf{ISS-08-005} — Swim unit and Pool \textbf{ID} logging drift risk: missing Pool \textbf{ID}/length-unit makes evidence inadmissible — Monitor Post-Launch
\item \textbf{ISS-08-007} — Logging burden overload risk: systematic missing fields can produce inadmissible evidence and \textbf{HOLD} outcomes — Monitor Post-Launch
\item \textbf{ISS-08-008} — Derived Cardio compliance misclassification risk: continuity and intensity anchors can be inferred rather than captured — Monitor Post-Launch
\end{itemize}

\section*{9.C.3 Readiness Statement Required}
\phantomsection
\addcontentsline{toc}{section}{9.C.3 Readiness Statement Required}

Program v1.0 readiness status: Not Ready.

Minimum Must Fix Pre-Deploy items required for release:

\begin{itemize}
\item \textbf{ISS-08-001} — Resolve pull-up gate feasibility timing so early phase gates do not require inadmissible/unschedulable pull-up evidence.
\item \textbf{ISS-08-002} — Enforce conditional underwater scheduling posture so underwater standards are not treated as mandatory when governance resources are unavailable.
\item \textbf{ISS-08-003} — Enforce admissibility computation for underwater exposure so \textbf{ST-UW-ADMISS} cannot be assumed without documented prerequisites.
\item \textbf{ISS-08-004} — Confirm and operationalize the long-ruck dependency readiness stack so decisive ruck gates are executable only when all required categories are available and stable.
\end{itemize}
